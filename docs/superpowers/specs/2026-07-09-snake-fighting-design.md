# Snake Fighting - 手机端贪吃蛇游戏设计文档

## 1. 项目概述

### 1.1 项目名称
Snake Fighting - 卡通可爱风格贪吃蛇游戏

### 1.2 项目目标
开发一款面向手机端的贪吃蛇游戏，采用卡通可爱视觉风格，支持多种游戏特色功能。

### 1.3 技术选型
- **框架**：React Native
- **渲染**：react-native-canvas
- **状态管理**：@reduxjs/toolkit
- **本地存储**：@react-native-async-storage/async-storage
- **音效**：react-native-sound

### 1.4 目标平台
- iOS 13+
- Android 8+
- 竖屏模式锁定

## 3. 核心游戏机制

### 3.1 游戏区域
- **网格系统**：20×20的网格（可根据屏幕大小动态调整）
- **坐标系**：左上角为(0,0)，向右为X正方向，向下为Y正方向
- **单元格大小**：根据屏幕宽度自动计算，保证正方形

### 3.2 蛇的行为
- **初始状态**：长度为3，位于屏幕中央，向右移动
- **移动逻辑**：每帧移动一个单元格，方向由滑动手势控制
- **生长规则**：吃到食物后长度+1，速度略微加快
- **死亡条件**：撞墙或撞到自己身体

### 3.3 食物系统
- **普通食物**：红色苹果，+10分
- **特殊食物**（随机出现）：
  - ⭐ 金色星星：+50分
  - 🚀 火箭：临时加速5秒
  - 🛡️ 盾牌：穿墙5秒
  - 🧲 磁铁：自动吸引附近食物5秒

### 3.4 难度递增
- **速度提升**：每吃5个食物，速度+1（最大速度有限制）
- **特殊食物频率**：分数越高，特殊食物出现越频繁

## 4. 视觉设计（卡通可爱风）

### 4.1 配色方案
- **主色调**：清新绿色（#4CAF50）+ 活力橙色（#FF9800）
- **背景**：浅绿色渐变（#E8F5E9 → #C8E6C9）
- **食物**：红色（#F44336）、金色（#FFD700）
- **UI元素**：白色卡片 + 圆角 + 轻微阴影

### 4.2 蛇的设计
- **头部**：圆形，带可爱的眼睛和微笑
- **身体**：圆角矩形，颜色渐变（从头到尾逐渐变浅）
- **动画**：移动时有轻微的弹跳效果

### 4.3 食物设计
- **普通食物**：苹果形状，带叶子
- **特殊食物**：带发光效果和简单动画

### 4.4 地图主题
1. **经典草地**：绿色背景 + 草地纹理
2. **沙滩冒险**：沙色背景 + 椰子树装饰
3. **冰雪世界**：蓝色背景 + 雪花效果
4. **太空漫游**：深色背景 + 星星闪烁

### 4.5 UI设计
- **开始界面**：大Logo + 开始按钮 + 排行榜入口
- **游戏界面**：左上角分数、右上角暂停按钮
- **结束界面**：得分统计 + 重新开始 + 分享按钮

### 5.3 游戏循环
```typescript
// 60fps游戏循环
const gameLoop = (timestamp: number) => {
  if (timestamp - lastUpdate >= gameSpeed) {
    updateSnake();      // 更新蛇的位置
    checkCollision();   // 检测碰撞
    checkFood();        // 检测吃食物
    render();           // 渲染画面
    lastUpdate = timestamp;
  }
  requestAnimationFrame(gameLoop);
};
```

### 5.4 滑动手势处理
```typescript
// 使用PanResponder处理滑动
const panResponder = PanResponder.create({
  onMoveShouldSetPanResponder: () => true,
  onPanResponderMove: (evt, gestureState) => {
    const {dx, dy} = gestureState;
    if (Math.abs(dx) > Math.abs(dy)) {
      // 水平滑动
      setDirection(dx > 0 ? 'RIGHT' : 'LEFT');
    } else {
      // 垂直滑动
      setDirection(dy > 0 ? 'DOWN' : 'UP');
    }
  },
});
```

## 6. 数据流和状态管理

### 6.1 Redux状态结构
```typescript
// 游戏状态
interface GameState {
  snake: {
    body: Position[];      // 蛇身坐标数组
    direction: Direction;  // 当前方向
    speed: number;         // 当前速度
  };
  food: {
    current: Food;         // 当前食物
    special: Food | null;  // 特殊食物
  };
  score: number;           // 当前分数
  highScore: number;       // 最高分
  isPlaying: boolean;      // 游戏状态
  isPaused: boolean;       // 暂停状态
  theme: string;           // 当前主题
}

// 用户设置
interface Settings {
  soundEnabled: boolean;   // 音效开关
  vibrationEnabled: boolean; // 震动开关
  controlType: string;     // 控制方式
}
```

### 6.2 数据流
```
用户滑动 → PanResponder → Redux Action → 更新状态 → Canvas重绘
    ↓
游戏循环 → 碰撞检测 → 更新分数 → 保存记录
    ↓
游戏结束 → 显示界面 → 保存最高分
```

### 6.3 本地存储
- **最高分**：每个主题单独记录
- **用户设置**：音效、震动、控制方式
- **游戏进度**：暂停时保存当前状态

## 7. 错误处理和边界情况

### 7.1 游戏逻辑边界
- **蛇撞墙**：游戏结束，显示得分
- **蛇撞自己**：游戏结束，显示得分
- **食物生成**：确保不会生成在蛇身上
- **方向冲突**：防止180度转向（如向右时不能直接向左）

### 7.2 性能优化
- **Canvas优化**：只重绘变化区域，而非全屏
- **内存管理**：及时清理不可见的特殊食物
- **帧率控制**：低端设备自动降低帧率

### 7.3 异常处理
- **存储失败**：静默处理，不影响游戏进行
- **音效加载失败**：跳过音效，继续游戏
- **手势识别失败**：保持当前方向不变

### 7.4 兼容性
- **屏幕适配**：支持不同尺寸的手机屏幕
- **系统版本**：支持iOS 13+和Android 8+
- **横竖屏**：锁定竖屏模式
